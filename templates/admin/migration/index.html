Migration Tips
  1. Import members first — all other member-linked records resolve by member_number or email. A portal login (role = member) is automatically created for each member that has an email; the default password is the member number.
  2. Then import savings — duplicates (same member + same month) are silently skipped.
  3. Then import loans — duplicate loan_number values are skipped. Purpose values (e.g. "Personal", "Education") are auto-normalised to the system's 5 standard types. Status aliases like "disbursed", "cleared", "repaid" are also mapped automatically.
  4. Then import repayments — link by loan_number (preferred), or by member identifier (uses the member's latest active loan). Loan balances and completion status are updated automatically.
  5. Finally import financial records — expenses, revenue, investments, and honorarium can be imported in any order.
  6. Use UTF-8 encoding when saving your CSV from Excel (Save As → CSV UTF-8).
  7. Date columns accept YYYY-MM-DD, DD/MM/YYYY, or MM/DD/YYYY.
  8. An import is processed in a single transaction — a critical file error rolls everything back. Row-level errors are reported and the rest of the file continues.
