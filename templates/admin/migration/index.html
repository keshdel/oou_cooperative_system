Deletes every member, saving, loan, expense, revenue, investment, and notification record. Your admin account and system settings are not affected. Use this to reset after a test import.

Migration Tips
  1. Import members first — savings, loans, and honorarium link to members by member_number or email.
  2. Then import savings — duplicates (same member + same month) are silently skipped.
  3. Then import loans — duplicate loan_number values are skipped.
  4. Finally import financial records — expenses, revenue, investments, honorarium can be imported in any order.
  5. Use UTF-8 encoding when saving your CSV from Excel (Save As → CSV UTF-8).
  6. Date columns accept YYYY-MM-DD, DD/MM/YYYY, or MM/DD/YYYY.
  7. An import file is processed atomically — if a critical error stops the file, no rows are saved. Row-level errors are reported but the rest of the file continues.
